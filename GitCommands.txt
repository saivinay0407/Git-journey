=== WORKFLOW FOR REPEATED CHANGES ===
When you make changes multiple times, you need to git add before each commit:

1. Make changes to files
2. $ git add . (add changed files to staging area)
3. $ git commit -m "commit message"
4. Make more changes
5. $ git add . (add updated files again)
6. $ git commit -m "next commit message"
7. Repeat steps 4-6 as needed

EXAMPLE:
$ git add .
$ git commit -m "first change"

$ git add .
$ git commit -m "second change"

$ git add .
$ git commit -m "third change"

NOTE: The staging area lets you verify what's going to be committed before you commit it.

Reset in git is used to remove the changes of the latest commits and has 2 main functionalities
1. hard: it will remove the commit id and all the data attached with that commit id
2. soft: it will remove the commit id, the data in file still remain same

$ git reset --hard HEAD~1
$ git reset --soft HEAD~1
